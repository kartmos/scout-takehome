# Exact corrections

## 1. Bounded route labels

1. After `next.ServeHTTP`, use `r.Pattern` for matched routes. When it is empty, record a named constant such as `unmatched`, never `r.URL.Path`.
2. Apply the bounded value to Prometheus labels. Access logs may also use the bounded fallback; never add raw path/query to metrics or logs.
3. Add a regression test issuing several distinct unknown paths/UUID-like paths and prove they produce one bounded route label and no raw path label.

## 2. Eviction metric

4. `scout_thumbnail_cache_evictions_total` must reflect real `Cache.evictLocked` removals without callers manually incrementing the metric.
5. Prefer a scrape-time monotonic collector (`CounterFunc` or equivalent) backed by `Cache.Stats().Evictions`, extending the injected stats callback cleanly. Do not introduce a circular dependency, global registry, racy setter, or double counting.
6. Keep cache bytes/entries gauges working. Update the metric test to cause an actual cache eviction and verify the gathered eviction value; do not call `.Inc()` directly in the test.

## 3. Strict cache-key safety

7. Validate every key accepted by exported cache operations before any index/filesystem access. Accept exactly the canonical task-010 SHA-256 form: 64 lowercase hexadecimal characters.
8. Reject empty, short/long, uppercase, non-hex, slash, backslash, absolute, `.` and `..` values with a safe error. No rejected value may create/open/remove a path or invoke generation.
9. Strengthen `TestCache_PathSafety` to prove the call fails, generator is not called, and no file appears outside or inside the cache root.

## 4. Correct coalesced hit/miss accounting

10. Define cache hit/miss per incoming request: a request that finds a committed entry on its initial lookup is a hit; every request that initially finds no entry is a miss, including followers coalesced behind one singleflight leader.
11. Generation metrics still count the leader's actual generation once. Do not count coalesced followers as generations and do not introduce a race around a closure-local `wasMiss` flag.
12. Return or propagate an explicit cache outcome from `Cache.GetOrCreate` (or an equivalent race-safe design) so `Service` records one hit or miss per request.
13. Add a deterministic concurrent test: N simultaneous cold requests produce N misses, zero hits, and exactly one generation; a subsequent warm request adds one hit and no miss/generation. Use channels/atomics, not sleeps.

## 5. Startup index cap and disk budget

14. Files beyond `maxIndexSize` must not be silently ignored while remaining as unaccounted valid cache entries. After startup, every valid cache-owned JPEG is either indexed/accounted or safely removed; indexed total remains within configured `maxBytes`.
15. Avoid loading an unbounded directory listing solely to enforce the cap. Scan in bounded batches if needed. Never follow symlinks or touch paths outside the cache root.
16. Remove only files that match the cache-owned canonical filename format or stale temp format. Leave unrelated files untouched and exclude them from cache accounting.
17. Add a focused regression without creating 100,000 files: make the index limit injectable/internal-testable or extract a startup helper accepting a limit. Verify overflow valid entries are removed/rejected, accounted bytes match disk cache entries, and the byte budget is respected.
