# Required corrections

1. **Final-page label.** In `GalleryPage`, show “(last)” whenever fulfilled `currentData` has no `next_token`, including a final page containing exactly `PAGE_SIZE` items. Keep Next disabled from the same condition. Add a regression case for a full final page.
2. **Valid interactive markup.** Do not wrap the entire card body in a `button`. The existing composition can place flow content and a nested Retry button inside it. Keep `article`/caption semantics and provide one explicit accessible Open-photo button/action that is separate from Retry. Preserve the trigger element passed to the viewer for focus restoration, keyboard access, image fallback, and card styling.
3. **Production bbox path.** Do not keep tested coordinate helpers that production never calls. Refactor one small pure rectangle-transform helper so both card and viewer overlays use it for their actual SVG rectangle attributes (normalized `1×1` for cards, native photo dimensions for viewer). Keep SVG `preserveAspectRatio` responsible for contain letterboxing. Remove obsolete `computeContainedRect`/`bboxToCssPixels` exports and tests if they are no longer the rendering path. Update required bbox tests to exercise the helper/components actually used in production, including clamping, invalid boxes, card/native dimensions, and DPR independence.
4. **Malformed photo isolation.** A malformed API photo ID must not throw from `PhotoCard` and unmount the gallery. Catch candidate construction locally without setting React state during render, render a stable unavailable-image fallback, and avoid offering a retry that can never repair invalid metadata. Keep ordinary network image failure retryable. Replace the false test at `GalleryPage.test.tsx` that overwrites the malformed fixture with a real assertion that sibling cards/gallery remain rendered and no uncaught render error occurs.
5. **Confidence blur.** Commit a valid confidence draft on blur as well as Enter/Apply. Avoid duplicate state changes/requests when clicking Apply after blur or when the parsed value already equals Redux state. Preserve external-reset synchronization. Invalid/out-of-range input must not silently issue an invalid query; keep behavior accessible and deterministic.
6. **Responsive `sizes`.** Align the image `sizes` expression with the actual grid padding, gap, `minmax(220px, 1fr)`, and column thresholds. Export one named sizes constant beside thumbnail candidates and test representative one-, two-, three-, and desktop-column ranges. Do not change cache identity or add more image candidates unless the corrected sizes contract proves it necessary.
7. Keep the independent reviewer’s accepted behavior unchanged: `currentData`, cursor bounds, same-prediction backend filtering, all returned predictions displayed, viewer containment/navigation/focus restoration, public thumbnail URLs, loading/empty/error states, and bounded one-page rendering.
