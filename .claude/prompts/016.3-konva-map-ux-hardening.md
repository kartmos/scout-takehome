# Correction 3 — expanded mode uses draft selection

Separate the location being explored in the expanded map from the location already
filtering the gallery.

Use explicit concepts:

```text
appliedLocation        # current gallery radius filter, owned by GalleryPage
appliedHighlightedId   # current highlighted gallery card
draftLocation          # expanded-map candidate only
draftHighlightedId     # candidate photo when a real marker was chosen
```

Names may differ, but responsibilities must remain separate.

## Entering expanded mode

1. Initialize the draft from the currently applied location and highlighted photo.
   If no location is applied, start with no draft.
2. Opening expanded mode must not alter the gallery, radius filter, pagination, or
   highlighted card.
3. Render the draft radius prominently. If an older applied radius differs, it may
   remain as a quieter reference, but the two states must not be visually confused.

## Selecting inside expanded mode

1. Clicking a real Konva photo marker updates only `draftLocation` to that photo's
   exact database `x,y` and sets `draftHighlightedId` to that photo ID.
2. Clicking empty map space updates only `draftLocation` to the converted world
   coordinate and clears `draftHighlightedId`.
3. Activating a row in the accessible marker list performs the same draft update as
   clicking that marker.
4. Draft selection redraws the 3 m circle and preview count immediately but does
   **not** filter the main gallery, change the applied location indicator, highlight
   a gallery card, or close the dialog.
5. Pan/drag above the existing movement threshold never changes the draft.
6. Class/confidence changes continue to update map data and the draft preview count.
   Do not silently apply the draft during a query refresh.

# Correction 4 — Apply/Cancel action bar

Add a persistent action bar at the bottom of expanded mode:

```text
[ Cancel ]    6 photos within 3 m    [ Apply location ]
```

1. `Apply location` is disabled while no draft position exists.
2. The preview count is derived from the current bounded map data, shared class/
   confidence filters, and the existing inclusive 3 m near predicate. It is a
   preview only until Apply.
3. On `Apply location`:
   - commit `draftLocation` through the existing applied-location callback;
   - commit `draftHighlightedId` through the existing photo-highlight callback;
   - close the native expanded dialog cleanly;
   - switch mode to `compact`;
   - collapse the marker list;
   - let the existing gallery derive near photos, hide normal pagination, show the
     location chip, and outline the chosen card when a marker was selected.
4. Applying an empty-space location commits the radius filter with no highlighted
   photo.
5. Applying the same already-active location is allowed and still returns to
   mini-map; do not require artificial coordinate changes.
6. `Cancel`, `Return to mini-map`, and Escape discard draft changes, preserve the
   previously applied location/highlight exactly, close the dialog, and return to
   compact mode.
7. `Hide map` from expanded mode also discards the draft and moves to hidden mode
   without altering the previously applied gallery filter.
8. Avoid duplicate native-dialog `close`/React-mode transitions. Use one clear
   close reason/state path so `onClose` cannot overwrite Apply with Cancel behavior.
9. Focus after Apply/Cancel returns to an appropriate mini-map control without
   focusing an unmounted expanded element.

# Correction 6 — coordinate readout beside zoom controls

Place a stable coordinate readout immediately to the left of the `+`, `−`, and
Reset buttons:

```text
x 12.5 m · y 8.0 m    [ + ] [ − ] [ Reset ]
```

1. In expanded mode display `draftLocation` coordinates.
2. In compact mode display the currently applied location coordinates.
3. With no relevant location display `Location not selected`.
4. Marker selection uses exact `photo.x/photo.y`. Empty-space selection uses the
   inverse stage-to-world result.
5. Format to one decimal place with tabular numerals and enough fixed/minimum width
   to prevent zoom controls jumping horizontally.
6. Coordinates must always remain within `0..40 m`; do not display stale or
   out-of-range pointer values.
7. Update only after an actual click/list selection, not continuously on pointer
   hover or during pan.
8. Use semantic status/output text. If `aria-live` is used, keep it polite and avoid
   duplicate announcements during render/resize.
9. `Clear location` returns the compact readout to `Location not selected`.
