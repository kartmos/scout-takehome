# Exact corrections

## Shared generation lifetime

1. A singleflight leader's request context must control only that caller's wait, not the shared generation executed for all followers.
2. Give `thumbnail.Service` an explicit non-nil generation-lifetime context supplied at construction. Production must pass the process/server signal context, so shared generation survives an individual client disconnect but stops during server shutdown.
3. Move `signal.NotifyContext` creation in `main` early enough to construct the service with it. Preserve signal handling, startup failures, `defer stop`, graceful shutdown timeout, and exit codes.
4. Inside `Service.Get`, continue passing each request context to `Cache.GetOrCreate` so a cancelled caller returns promptly. The generation closure must call `Generator.Generate` with the service lifetime context, never the leader request context.
5. Do not use an immortal `context.Background()` or merely `context.WithoutCancel(requestCtx)` in production. Do not merge contexts with helper goroutines or create one goroutine per request.
6. Update internal service constructors/call sites consistently. A test-only/default constructor may use an explicit caller-provided context; avoid hidden lifetime semantics.

## Deterministic regressions

7. Add a concurrency regression with channels/atomics and no sleeps:
   - two cold requests for one key join the same flight;
   - cancel the leader request after generation starts;
   - the leader returns cancellation promptly;
   - the live follower succeeds from the same one generation;
   - exactly one generation/commit occurs and no goroutine remains blocked.
8. Add a lifecycle test proving cancellation of the service/server generation context aborts in-flight generation and wakes live waiters with a cause matching cancellation.
9. Run these under the race detector. Preserve existing cold-burst accounting and duplicate-suppression tests.

## Miss semantics documentation

10. Keep the task-012.1 definition: every request whose initial lookup finds no committed entry is a cache miss, including coalesced followers; only the leader increments generation metrics.
11. Document this explicitly on `ServiceHooks.OnCacheMiss` and in the Prometheus cache-miss help text so operators do not interpret misses as generation count. Do not add another metric or change accounting code unless needed for consistency.

## Small cleanup

12. In `handleGetThumbnail`, call `r.URL.Query()` once, store the result, validate it, and pass the same values to `thumbnail.ParseRequest`.
13. Remove the unused `time` import and `var _ = time.Now` shim from `thumbnails_test.go`. Do not add a fake use or unrelated timeout test.
