# “Near” definition and location filter

Use an explicit named constant:

```text
NEAR_RADIUS_METRES = 3
```

1. A click/tap on empty stage converts the pointer position back to world `x,y` and
   selects that location if it is inside `0..40`.
2. Clicking a marker selects its exact photo `x,y` without also triggering the
   background handler.
3. A photo is near when Euclidean distance is inclusive:

   ```text
   hypot(photo.x - selectedX, photo.y - selectedY) <= 3
   ```

4. Draw the selected location and a translucent 3 m radius circle on the Konva
   layer. Its world radius scales with the map while stroke/centre handles remain
   legible.
5. Store selected world `{x,y}` in `GalleryPage` local state or an equally narrow
   map/gallery owner. Do not add it to Redux, URL, localStorage, API, or photo data.
6. Derive `nearPhotos` from the same bounded, class/confidence-filtered map
   `currentData` using the pure helper. This produces the required intersection:

   ```text
   real spatial radius ∩ class filter ∩ minimum-confidence filter
   ```

7. When location selection is active, render `nearPhotos` through existing
   `PhotoCard`, hide normal cursor pagination, and preserve the saved normal page/
   cursor state for restoration.
8. Show a visible indicator above the gallery, for example:

   ```text
   [ Near x 12.5 m, y 8.0 m · 6 photos ]  [ × Clear location ]
   ```

   `Clear location` is a real non-underlined button and clears only spatial
   selection. Existing `Clear filters` continues to clear class/confidence.
9. If shared filters make the selected radius empty, keep the location selection,
   redraw counts/markers, and show a useful empty state with clear-location/filter
   actions.
10. Clearing location restores the exact prior normal paginated gallery state.
