# Required corrections

## 1. Seed container must execute the seed binary

The backend image has `ENTRYPOINT ["scout-api"]`, while local Compose currently supplies `command: ["scout-seed"]`. Compose appends that command to the entrypoint and therefore attempts to run `scout-api scout-seed`.

- Override the seed service entrypoint with exec form `scout-seed` and remove any misleading command that becomes an API argument.
- Keep the API service on the image's default entrypoint.
- Confirm the rendered Compose model shows the seed entrypoint and does not render `scout-api scout-seed`.

## 2. Make Nginx behavior match actual Vite output

The current regex accepts only an eight-character lowercase hexadecimal filename segment, but Vite emits base64-like mixed-case hashes. Replace it with a path-based immutable rule for generated `/assets/` content rather than guessing a hash alphabet.

- Serve `/assets/` with long-lived immutable caching.
- Serve `index.html` with revalidation/no-cache and retain SPA fallback.
- Account for Nginx `add_header` inheritance: locations that set their own cache headers must not silently lose CSP and the other security headers. Use a small include or repeat the complete security-header set consistently.
- Keep query-free access logging, bounded proxy timeouts/body size, and prefix stripping for `/api/`.
- Disable proxy response buffering for API responses if the README/config claims streaming behavior.
- Do not add an API key in Nginx and do not proxy presigned object bytes through the app.

## 3. Normalize or reject root-relative `/` safely

Frontend URL validation currently accepts `/` and normalizes it to `/`; request builders then concatenate `/photos`, yielding `//photos`, which is protocol-relative in browser URL handling.

- Prefer normalizing root `/` to the empty same-origin prefix so generated routes are exactly `/photos`, `/healthz`, and similar single-slash paths. Rejecting `/` is acceptable only if the user-visible validation reason is explicit and all callers remain coherent.
- Preserve `/api` support and all existing rejection rules for protocol-relative URLs, traversal, backslashes, credentials, query, and fragment.
- Add focused regression tests for `/`, `/api`, trailing slash normalization, and generated request paths. Test behavior rather than only the validation helper.

## 4. Make MinIO health work with the configured port

Local MinIO is configured to listen on `SCOUT_MINIO_API_PORT`, but `mc ready local` may target the default port and can fail when the configured port differs.

- Make the healthcheck explicitly target the effective configured API endpoint/port, without printing credentials or signed URLs.
- Preserve the `minio.localhost` network alias and matching public endpoint used by both the host and seed container.
- Verify the rendered Compose model with a non-default port as well as the default.

## 5. Remove floating project tags and keep realistic memory headroom

- Replace every project image reference using `:latest` with one concrete versioned tag, for example `scout-api:0.1.0` and `scout-web:0.1.0`, consistently in Compose and README.
- Do not invent unavailable upstream patch tags. If the Docker daemon/registry cannot validate a more specific upstream tag, retain the currently compatible upstream family tag and report that exact limitation; never use `latest`.
- Production API currently leaves only about 24 MiB between `GOMEMLIMIT=360MiB` and a `384m` container limit. Lower the Go memory limit to leave practical native/runtime headroom, for example about `320MiB`, while keeping the declared API+web budget within the documented target.
- Correct any comment that describes the disk thumbnail cache as memory headroom.

## 6. Add one real offline public-presign regression if absent

Fake-client tests prove routing decisions but do not prove the constructed MinIO client emits the configured public scheme/host.

- Add a focused, network-free test that constructs the real adapter/client with distinct internal and public endpoints and asserts a presigned URL uses the expected public scheme and authority.
- The test must not require MinIO, Docker, DNS, or network access and must not expose credentials.
- Keep internal read/check behavior covered separately.
