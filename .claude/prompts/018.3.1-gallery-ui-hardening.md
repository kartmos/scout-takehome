# Correction 3 — confidence affects displayed detections

## Problem

With ten predictions ranging from `47–54%`, selecting minimum confidence `50%`
still shows `10×` and `47–54%` because the response intentionally contains all
predictions and the frontend groups the unfiltered array.

## Required behavior

1. Define one shared display rule using raw confidence values:

   ```text
   visiblePredictions = minConfidence == null
     ? predictions
     : predictions.filter(prediction => prediction.confidence >= minConfidence)
   ```

2. Comparison is inclusive and must happen before percentage rounding. A raw
   `0.499` prediction is excluded by a `0.5` threshold even if its displayed rounded
   percentage would be `50%`.
3. Pass the active `minConfidence` from `GalleryPage` to both `PhotoCard` and
   `PhotoViewer`. Keep it a ratio; do not convert the prop to an integer percentage.
4. In `PhotoCard`, use `visiblePredictions` consistently for:
   - grouping;
   - group counts;
   - minimum/maximum displayed confidence;
   - deciding which class rows exist;
   - rendered bbox rectangles.
5. Example: ten Powdery Mildew predictions in `47–54%` may become
   `5× Powdery Mildew 50–54%` after a `50%` threshold, depending on the actual raw
   values. Never keep `10×` or a `47%` lower bound if those detections were removed.
6. In `PhotoViewer`, apply the same threshold to the fresh photo response before
   grouping, numbering, listing, highlighting, and drawing/toggling bboxes. Numbers
   remain deterministic and contiguous for the visible set.
7. Preserve class-filter semantics:
   - with `All`, all above-threshold bboxes are normally emphasized;
   - with a class selected, that class remains prominent and other classes above
     the confidence threshold remain visible but dimmed;
   - below-threshold bboxes are hidden regardless of class;
   - with no confidence threshold, card and viewer behavior remains as before.
8. Do not mutate API response arrays. Keep the filtering/grouping helper pure.
9. If a successful response has predictions but none pass the active display
   threshold, use wording such as `No detections at this confidence` rather than
   claiming the source photo has no detections.
10. Viewer per-detection visibility state must continue to reset safely on photo
    navigation. It must refer only to the threshold-filtered, currently numbered
    set and must not leak stale indices across photos or threshold changes.
