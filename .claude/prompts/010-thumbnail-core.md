Implement production-only thumbnail request parsing and bounded JPEG generation. Derive deterministic dimensions/cache identity, prevent unsafe memory use and upscaling, stream originals from object storage, and limit generation concurrency. Do not add an HTTP endpoint, cache, metrics, or tests.

# Production contract

1. Define an HTTP-independent request type/parser for raw `width`, `dpr`, `quality` values, callable by task 011.
2. Contract: required base-10 `width` `1..2048`; optional finite `dpr` default `1`, range `1..3`; optional base-10 `quality` default `80`, range `1..100`. Reject empty supplied values, whitespace, inappropriate signs, NaN/Inf, overflow, and unknown inputs.
3. Convert `width × dpr` to integer requested pixels with one documented rounding rule and validate multiplication before conversion.
4. Resolve dimensions from actual source width/height, preserve aspect ratio, never upscale, keep dimensions positive, and enforce a named output-pixel ceiling.
5. Canonical identity depends only on canonical photo UUID, resolved dimensions, quality, JPEG format, and a named generator-version constant. Equivalent representations share one key. Expose a stable hashed key; never expose raw parameters as filesystem paths.
6. Define a small generator API accepting context, `domain.Photo`, parsed request, and `io.Writer`, returning immutable representation metadata plus error.
7. Validate UUID and persisted dimensions before storage access.
8. Open the original only after validation, always close it, and preserve the primary failure while wrapping close failure when appropriate.
9. Read JPEG configuration first. Reject non-JPEG/corrupt input, source dimensions above named width/height/pixel limits, and dimensions inconsistent with database metadata.
10. Decode from the storage stream, resize with a quality-oriented `x/image/draw` scaler, and encode JPEG directly to the writer. Do not retain generated bytes in a package/global buffer.
11. If output equals source dimensions, avoid a resize allocation while still encoding at requested quality.
12. Add `SCOUT_THUMBNAIL_GENERATION_CONCURRENCY`, default `1`, allowed `1..2`. Acquire a context-aware semaphore before opening/decoding and hold it through encoding.
13. Check context around expensive phases and wrap the source reader so cancellation can stop reads. Preserve cancellation/deadline matching.
14. Define safe typed errors for invalid request, source not found, unsupported/corrupt image, and internal generation/storage failure. Preserve causes; never expose URLs, credentials, paths, or response bodies.
15. Use named constants, document memory rationale, and introduce no background workers or mutable global state.
